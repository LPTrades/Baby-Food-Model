## Version Update - 2026/05/28
## Changes

* CISD level detection: orange line at the open of the first candle in the most recent consecutive bearish run inside the swing high → swing low range
* Bearish IFVG detection using the FVG formed by that same consecutive run
* Entry arrow (long) fires when price closes above both the CISD level and the IFVG in the same candle
* Mirror short setup: swing high break → consecutive bullish run → CISD + IFVG → short entry arrow
* Stop placed at lowest open/close across all 6 candle bodies (CISD run + FVG candles), bodies only
* TP set at 1.5R from entry
* Diagnostic overlays: yellow delivery count dots, green/red triangles per bar showing bear/bull read, state dot below bars

## Bugs Fixed

* Broken swing lines bumped to width 2 for visibility
* Stop-capped logic removed from backtester; replaced with clean wide-stop skip (skipped trades counted separately in stats output)
* CISD not firing — diagnosed via delivery_count dots and candle-read triangles; root cause was reset/detection logic

## Reason For Changes

* Previous FVG-only and IFVG chain implementations lost money in backtesting; CISD + IFVG is the current model to test
* Diagnostic overlays added to make it possible to verify what the script is actually reading vs what price looks like visually

## Observations

* Figure out what stop is too wide
* Delivery count diagnostics revealed the CISD detection issue was behavioural (reset timing) not structural. It's dynamic and the script needs to account for new runs forming new lower lows in the detection range.
