# Daily Research Log - 2026-05-23

## Objective

Extend the v0.6 Pine Script ("Swing - First FVG Only") to detect an **Inverse Fair Value Gap (IFVG)** alongside the existing FVG, both firing off the same 5m swing sweep / break. Detection and visualization only — no entry, stop loss, take profit, or trade management this session.

---

## What I Worked On

- Translated the IFVG concept into Pine: an opposite-side gap that price trades back into and closes through, flipping its role.
    - Long: a **bearish** gap that price closes **above** → flips to a **bullish IFVG**.
    - Short: a **bullish** gap that price closes **below** → flips to a **bearish IFVG**.
- Built the IFVG detector first as a standalone (replacing the FVG), then revised it, then re-added the FVG so both run in parallel.
- Decided the IFVG should track the **last** opposite gap before the reversal, not the first.
- Set up two independent detectors off the same break — separate state, separate boxes, both reset on the next sweep.
- Gave the boxes distinct styling (FVG solid border / lighter fill, IFVG dashed border / heavier fill).

---

## Problems

- **First vs last gap.** My initial instruction was ambiguous — I had to clarify that the IFVG keeps the _last_ bearish gap and inverts that one, not the first.
- **Same-bar form-and-flip risk.** If detection and flip logic both run on one bar, a gap could appear to register and invert simultaneously, which isn't real. Required ordering the flip check before the gap-tracking update.
- **State collision.** The original IFVG build reused the v0.6 `fvgFound` flag, which would have made the two detectors block each other once both were running.
- Entry price for this model is still undefined (open question carried over from the Oracle doc).

---

## Successes

- IFVG detection working: tracks the most recent opposite gap and draws the flipped box when price closes through the edge.
- FVG and IFVG now run as **two fully independent detectors** off the same break, both visible at once.
- State cleanly separated (`fvgFound` vs `ifvgFound`, separate boxes) so neither detector interferes with the other.
- Boxes are visually distinguishable on the chart.
- v0.6 swing / break / line logic preserved intact.

---

## Insights

- The **last** opposite gap before a reversal is closest to price and therefore the first to get inverted — so tracking the latest gap (not the first) matches how price actually flips an IFVG.
- FVG and IFVG look for **opposite gap polarities** (continuation vs. the gap that inverts), so they rarely land on the same candles and stay visually distinct.
- Parallel detectors need fully separate state, or they silently block each other.
