# Daily Research Log - 2026-05-24

## Objective

- Turn the FVG + IFVG indicator from detection-only into a tradable model: add an entry signal, then a stop and a target.
- Build a Python backtester for the new model so the logic can be validated on historical NQ data.

## What I Worked On

- Added entry arrows to the Pine script — fire on the open of the candle after both FVG and IFVG confirm, order-agnostic, 1m-only, direction from mode.
- Attached the stop: lowest open/close (long) / highest open/close (short) across all 6 gap candles (FVG C1/C2/C3 + IFVG C1/C2/C3), bodies only, wicks ignored.
- Added a 1.5R take profit; entry/SL/TP drawn as lines. Finalized as the v0.8 Pine script.
- Built the Python backtester to match v1.0: parallel FVG + IFVG detection, next-bar-open entry, 6-candle body stop, 1.5R TP, session-gated breaks, exits held until TP or SL. Same data paths and stats block as the old backtester.
- Diagnosed and fixed the slow swing-build stage by vectorizing it.
- Kept the version docs current (v0.8, v1.2) and wrote up the AI dev log.

## Problems

- Pine compile error: "NA type cannot be assigned to a variable defined without type keyword." Caused by comma-joined assignments on one line. Pine doesn't support that syntax.
- "Building swing levels" was painfully slow — two pure-Python loops scanning every 5m bar across the full file.
- Had to be careful that swings weren't accidentally session-filtered, which would have missed valid breaks of pre-session swings.

## Successes

- Full model now defined end to end: entry, stop, and target all set the moment the setup confirms — no discretionary decisions left.
- Compile error fixed by splitting each assignment onto its own line; script compiles clean.
- Swing build vectorized with pandas shifts — identical output, near-instant even on large data.
- Backtester runs clean end to end on a synthetic smoke test; produces the same stats format and 20 random 2026 examples as before.

## Insights

- The new entry is a market fill at the open after confirmation, NOT the old touch-the-FVG trigger — a fundamentally different mechanic that changes how the backtester has to work.
- Body-based stops mean risk-per-trade in points is no longer uniform; tight gaps produce tight stops that may get wicked out more often. Worth watching in the real backtest.
- Swings must be detected across all hours; session filtering belongs at break detection, not swing build.
- The one-bar-delay open fill introduces real-world slippage worth measuring against the confirmation close.
